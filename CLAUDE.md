## Guidelines Shared by All Projects

When users ask you to do something (even if it's a simple task), don't do it immediately. You MUST first look up the relevant guidelines listed below, and then execute the task following the guidelines. If the guidelines refer to other files, you MUST also read the referenced files.

- For adding new components, follow the instructions in `guideline/workflow.md`.

- For coding style, follow the instructions in `guideline/coding-style.md`.

- Before making ANY git commit, read and follow `guideline/git-commit.md`.

- For analyzing the results of training, refer to the structure of the logs in `guideline/logs-structure.md`.

-For any questions, eapecially the first question, read 'guideline/focus_on.md' to find out which parts of code we are focusing on.
